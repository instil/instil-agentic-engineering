Delivery Framework

Agentic

Software delivery is shifting. AI agents can now take on entire tasks, from implementing features to reviewing code and monitoring production. This document describes how Instil is adapting its delivery process to pair engineers with specialised agents at every stage, keeping engineering judgement at the centre while using agentic development to increase the pace, breadth and rigour of what teams can deliver.

The Delivery Team

Engineers and agents work in parallel at every stage, and what each focuses on shifts as the project moves forward

E
Engineers

Engineers own the codebase, the technical direction, stakeholder alignment and the accountability for what ships. They run discovery, make architecture decisions, evaluate agent output and steer implementation toward the intended design. Every critical decision has an engineer behind it, and all agent output goes through engineering review before it moves forward.

A
Agents

Agents handle the implementation and analysis work across the full delivery cycle. In the early stages that means research, requirements capture and prototyping. As the team moves into build, agents take on first-pass implementation, code review, testing and security analysis. In production, they manage deployment, monitoring and incident response.

How This Plays Out in Practice

What changes when engineers and agents work side by side across real projects

Where engineers spend their time

Agents handle the volume of code generation, test writing and routine analysis. Engineers spend less time on mechanical work and more on the problems that require judgement. The engineering skill set underneath is the same one it has always been.

The codebase ramp is shorter

Agents that can ingest, navigate and explain a codebase change onboarding dramatically. Engineers still need to build real understanding of the system, but agents accelerate that process — meaningful contributions come sooner.

More gets reviewed, not less

Output per engineer is increasing, which means review matters more than ever. Every change goes through review from multiple angles: code quality, test coverage, security and alignment with requirements.

Security throughout

Threat modelling agents work during planning, security analysis agents run static and dynamic checks throughout the build, and monitoring agents watch for issues that only surface at runtime. Security runs throughout because agents make it practical to do so.

What We Hold To

The principles that guide how we work with agentic tooling

The right approach for the right problem

Not every task suits full delegation. Engineers develop a feel over time for what to hand off and what to keep. Some tasks suit complete agent delegation, others need direct human work. Most sit somewhere in between — knowing which is which is an engineering skill that compounds with experience.

Continuous learning is built in

Engineers who work with agents daily build practical knowledge that no course can teach, i.e. when to delegate, when to direct and when to do it by hand. That learning is supported through pairing, mentoring and communities of practice, but the deeper understanding comes from applying it.

Honest about where it doesn't work

Agentic tooling doesn't suit every task equally. Development agents can struggle with complex legacy systems and deep domain context. But even where implementation agents don't help, review, testing and security agents often still add value. The skill is knowing which agents add value where.

AI-generated output can get you 80% of the way there. The last 20% is where engineering judgement matters most; spotting what's subtly wrong, understanding why, and knowing how to fix it. That skill comes from building software, and it only becomes more important in this model.
When engineers have agents working alongside them, teams can take on more ambitious problems, the scope of what's achievable grows, and engineers spend their time on the parts of software delivery that are most consequential: figuring out what to build, designing it well, and making sure it actually works for the people who use it.
